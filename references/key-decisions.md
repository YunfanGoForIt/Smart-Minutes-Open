# 关键决策（# 关键决策）

**先判断会议性质**：决策型（与会者就某事拍板、达成共识、确定方案/方向）vs 分享/讨论型（纯信息分享、观点交流，无任何共识达成）。

**关键：区分"个人建议"与"会议共识"**：
- **个人建议/观点**（不算决策）：分享者个人的经验、建议、某公司做法介绍（如"建议大家补单测""我认为三岗合一更好"）。
- **会议共识/决策**（算决策）：与会者**共同认可并打算落实**的方向、方案、安排（如"双方一致确定三级产品矩阵架构并计划暑期细化""决定暂缓与 X 合作""时间定为下周一"）。即便没有正式"拍板"仪式，只要讨论中达成了**可落地的共识方向**，就应作为决策输出。

## 6a. 若有会议共识/决策——分主次输出

若有多个决策，**分主次**：选 1 条最重要的作为"关键决策"用完整三要素展开；其余作为"其他决策"用简列格式（一行一条，不展开三要素）。这对应原版飞书纪要的"1 主决策 + N 其他决策"结构。

**主决策选取**：选**影响全局/贯穿全篇**的核心决策作主决策（如"考试范围与题型""确定三级产品矩阵架构"这类定义性决策），仅把次要行政安排（打分座位、报告顺序、礼盒发放）降为其他决策。**不要把核心信息降级**——若一场会议的全局性安排（范围/形式/方向）被降为其他决策，会弱化纪要主线。

**关键决策**（最重要的 1 条，完整三要素）：

```
- **{决策一句话}**

  - **问题**：{要解决的问题/背景}
  - **讨论方案**：
  
    - {方案/观点 1}
    - {方案/2}
  - **决策依据**：{为什么这么定}
```

**其他决策**（简列，每条一行）：

```
- **其他决策**：

  1. {决策一句话}
  2. {决策一句话}
  3. {决策一句话}
```

若只有 1 条决策，就只输出关键决策、无"其他决策"段。

## 6b. 若确无任何共识（纯分享/信息同步）——保留章节但标注

**慎用 6b**：分享/讨论类会议往往仍有"建议性决策"——分享者给出的**核心方法论/落地建议**（如"先数字化再 AI 化""分层分级推进"）应作为**建议性决策**用 6a 三要素格式呈现（问题/讨论方案/决策依据），**不要一律判"无决策"跳过结构化格式**。

只有当会议**全程无任何方向性结论、无核心方法论提炼**（纯闲聊/纯信息同步）时，才保留 `# 关键决策` 章节并写：

```
本次会议为分享/讨论类会议，未形成组织层面的关键决策。会议核心内容为：

- **{嘉宾/主题 1 的分享要点}**：{2-3 条}
- **{嘉宾/主题 2 的分享要点}**：{2-3 条}
- （圆桌议题若有，列出讨论的主要议题与各方观点）
```

> 判断标准：能从分享中提炼出"应该怎么做"的方法论/建议→用 6a；纯粹是"发生了什么/聊了什么"无任何"应该"→用 6b。

> 慎用 6b：聚餐讨论、规划商讨类会议往往仍有共识方向，应优先用 6a 结构化输出，不要轻易判"无决策"。
