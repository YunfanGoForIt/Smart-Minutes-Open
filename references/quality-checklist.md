# 全局质量要求与自检清单

## 全局质量要求

1. **忠实优先**：所有内容可在文字稿中找到依据。人名、数字、结论不得臆造。口语可润色但不可改义。
2. **结构还原**：`<title>`、引用块、`<grid>/<column width-ratio>`、`<whiteboard>`、`# 章节`、`- [ ]`、`<cite>` 等飞书特有结构要正确出现。
3. **简要**：画板与要点都要"简要"，避免长篇大论；信息密度高、无水分。
4. **发言人**：默认用"说话人 1/2"，不臆造姓名；引用原话时可用"说话人 1 表示……"。
   - **若要用人名**（如某嘉宾自我介绍、被他人明确称呼），**必须先做身份映射校验**：从文字稿提取所有"我是 XX""你是 XX""XX 说""XX 是我们的 CTO"等自报/他指语句，建立 speaker→姓名 映射。**只有当某 speaker 自己说了姓名或被当场指认，才能把该姓名挂到该 speaker**。仅在他人转述中出现的第三方（如"毕阳是我们的 CTO"是说话人 A 转述的第三人），**绝不可**挂到任何 speaker 上——这种情况该姓名按"第三方提及"处理，不作为 speaker 身份。映射不确定时一律回退用"说话人 X"。
5. **主题命名**：`{主题}` 从文字稿内容归纳，≤20字，名词性短语。
6. **日期**：取自第 1 行时间，格式"YYYY年M月D日"。
7. **术语还原校验**：文字稿含大量 ASR 音译/误识术语。还原为标准术语时**必须基于上下文验证技术正确性**——若不确定正确英文，**保留原音译写法**（如写"扣扣2分解"而非猜测为 Cholesky），切忌把不相关的英文术语硬套上去。
8. **术语/专名溯源防幻觉**：任何英文专名、效应名、术语（如"ISO curve""Staff 效应"）若在文字稿中找不到对应的发音/出现依据，**不得自行补充**——保留音译或标注"（原文未明确）"。宁可少写，不可编造文字稿里没有的概念。
9. **分值/比例防幻觉**：原文说"占 X 分""X 分"（满分制分值）时，必须写"占 X 分"，**不得改写为"占 X%"**；只有原文出现"%"或"比例/占比"字样时才能写百分比。数字与单位必须与原文一致，分值≠比例。
10. **固定搭配/术语校验**：对"概算/预算/核算/决算"等四字固定搭配、"协方差矩阵特征值分解/KL 变换"等专业术语，输出前校验是否为标准写法。ASR 混淆的缩写统一取首次出现的写法或加"（缩写待核）"标注。

## 自检清单

按以下顺序逐项检查（每条要点/待办/决策须能回溯到文字稿原文）：

### 1. 画板自检
- [ ] 画板 JSON 是否符合结构规范（含文字节点 `height=fit-content`、frame 显式 `gap/padding`、配色取自经典色板）？
- [ ] `画板.png` 是否无文字截断/节点重叠（`--check` 通过）？
- [ ] 写入是否执行或已标注降级？`<whiteboard token>` 是否用了真实 token（写入成功时）/ `WHITEBOARD_TOKEN` 占位（降级时）？
- [ ] 是否误把画板做成了"🖼画板内容"文字 grid（应改为 JSON 画板）？

### 2. 章节与结构自检
- [ ] 章节时间戳是否唯一且对得上文字稿？是否精确到秒？
- [ ] 闲聊/生活话题是否已合并到相邻主题或合并为 1 章？
- [ ] 章节是否覆盖到了文字稿最后一个说话段落？
- [ ] 第 1 行是否 `<title>` 包裹？
- [ ] 录音时间格式是否为 `{YYYY年M月D日}（{周X}）{HH:MM} - {HH:MM}（GMT+08）`？
- [ ] **每一个**一级章节（含金句时刻、相关链接）末尾是否都加了 `<readonly-block type="isv">`？

### 3. 内容忠实度自检
- [ ] 是否把个人建议误当决策/待办？
- [ ] 待办 cite 是否含 user-id 占位符 `USER_ID_PLACEHOLDER`？
- [ ] grid 是否只用于局部图文并排而非包整个总结？
- [ ] 正文对比表是否保留 grid 结构未退化 bullet？
- [ ] 金句是否含至少 1 条核心长原话？是否 3 条为佳（定调/方法论/升华各一）？
- [ ] 要点是否保留了生活/闲聊次要主题？
- [ ] 讲座类要点是否保留了 ~80% 原版子要点、按"第X块"叙事顺序？
- [ ] 是否存在数据幻觉（分值≠比例、术语溯源缺失、英文专名无凭据）？
- [ ] 会议性质判断是否正确？（影响章节粒度、决策写法、待办有无）
